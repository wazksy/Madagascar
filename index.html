    Madagascar

    Un framework CSS escrito en SASS que hará tu desarrollo frontend más sencillo

    Madagascar es un framework CSS escrito en SASS que sigue la notación BEM y que facilita el desarrollo frontend. Madagascar cuenta con un sistema de grillas sencillo y una paleta de colores que genera multidud de matices automáticamente gracias a SASS a partir de valores iniciales.

    Madagascar incluye multitud de clases que permiten aplicar estilos a una página agregando directamente clases en el archivo HTML sin la necesidad de escribir mucho código específico, siguiendo así varios paradigmas de desarrollo CSS. Esta tendencia puede considerarse una mala práctica, pues se crea así un código HTML menos semántico, más pesado y complejo, pero tiene la ventaja de que el código CSS es altamente reutilizable.

    Este framework está siendo escrito con el objetivo principal de desarrollar diseños para WordPress, por esa razón se tiene en cuenta que muchas de las clases puedan ser sobrescritas con valores obtenidos desde la base de datos de WordPress. También los componentes diseñados dentro de Madagascar tienen como objetivo poder ser utilizados como shortcodes para WordPress.

    Colores

    A partir de las variables de color definidas en el archivo _variables.scss SASS genera toda la paleta cromática.

    Mediante SASS también se definen automáticamente clases para background como .bck--theme-500, clases para bordes como .border--secondary-a-200 y clases para color de texto como .txt--grey-800.

    Los valores centrales de cada color tienen el valor 500 y las variaciones cromáticas se generan a partir de él, salvo en los tonos de gris.

    Grid & Layout

    El sistema de grillas de Madagascar utiliza una nomenclatura semántica y cuenta con un máximo de 4 columnas. Establece una separación entre columnas en px en función del valor de la variable gutter que encontramos en el archivo _variables.scss. Además es un sistema responsive ya que permite definir diferentes estructuras de columnas para cada ancho de pantalla utilizando clases específicas. Los tamaños de pantalla son xs, s, m, l y xl y sus valores se pueden modificar en el archivo _variables.scss.

    Para crear una estructura de grillas debemos agrupar todo el contenido dentro de una etiqueta con la clase .row, que añade reglas clearfix y establece márgenes negativos para evitar el uso de clases para la primera y la última columna o para columnas anidadas. A su vez, todo el contenido debe estar agrupado dentro de una etiqueta con la clase .wrapper, que establece un ancho máximo a partir del valor de la variable max-width que econtramos en el archivo _variables.scss y centra todo el contenido o agrupado dentro de una etiqueta con la clase .wrapper-full que utiliza todo el ancho de la pantalla.

    Las columnas se establecen mediante la clase .col y mediante clases del tipo .col--xs-one-fourth. Una misma columna podrá tener varias clases diferentes para que tenga comportamientos diferentes en cada ancho de pantalla. Dado que el sistema de grillas utiliza el paradigma mobile first, en primer lugar debemos establecer la clase para el ancho de pantalla más pequeña .col--xs-one-half y las clases que añadamos para otros anchos de pantalla superiores sustituirán los valores de la primera cuando sea necesario.

    También hay disponibles algunas clases para el tratamiento del layout. Por defecto las filas y muchos otros elementos no incorporan márgenes inferiores para la separación horizontal. Para ello deberá añadir la clase .margin--bottom al elemento.

    También podrá anular el margen o el padding inferior de un elemento con las clases .padding--no-bottom y .margin--no-bottom.

    Con las clases .border--right, .border--left, .border--bottom y .border--top puedes crear separaciones entre columnas o filas.

    Semántica

    Las etiquetas HTML tienen un valor o significado semántico dentro de la página. En el siguiente web de la organización Mozilla podemos encontrar un listado completo de las etiquetas HTML y su correspondiente función.

    Dentro de este documento recojemos las etiquetas HTML más importantes en relación con texto.

    Etiqueta Descripción Ejemplo
    a Representa un hiperenlace, enlazando a otro recurso. Enlace
    em Representa un texto enfatizado, como un acento de intensidad. Texto enfatizado
    strong Representa un texto especialmente importante. Texto importante
    small Representa un comentario aparte, es decir, textos como un descargo de responsabilidad o una nota de derechos de autoría, que no son esenciales para la comprensión del documento. Comentario aparte
    s Representa contenido que ya no es exacto o relevante. Contenido inexacto
    cite Representa el título de una obra. Título
    q Representa una cita textual inline. Cita textual
    dfn Representa un término cuya definición está contenida en su contenido ancestro más próximo. Término
    abbr Representa una abreviación o un acrónimo; la expansión de la abreviatura puede ser representada por el atributo title. Abbr
    data Asocia un equivalente legible por máquina a sus contenidos. (Este elemento está sólamente en la versión de la WHATWG del estandar HTML, y no en la versión de la W3C de HTML5). data
    time Representa un valor de fecha y hora; el equivalente legible por máquina puede ser representado en el atributo datetime. Fecha cualquiera
    code Representa un código de ordenador. Código de ordenador
    var Representa a una variable, es decir, una expresión matemática o contexto de programación, un identificador que represente a una constante, un símbolo que identifica una cantidad física, un parámetro de una función o un marcador de posición en prosa. Variable
    samp Representa la salida de un programa o un ordenador. Salida de un programa
    kbd Representa la entrada de usuario o usuaria, por lo general desde un teclado, pero no necesariamente, este puede representar otras formas de entrada de usuario o usuaria, como comandos de voz transcritos. Entrada de usuario
    sub y sup Representan un subíndice y un superíndice, respectivamente. SubíndiceSuperíndice
    i Representa un texto en una voz o estado de ánimo alterno, o por lo menos de diferente calidad, como una designación taxonómica, un término técnico, una frase idiomática, un pensamiento o el nombre de un barco. Texto alterado
    b Representa un texto hacia el cual se llama la atención para propósitos utilitaros. No confiere ninguna importancia adicional y no implica una voz alterna. Llamada de atención
    u Representa una anotación no textual sin-articular, como etiquetar un texto como mal escrito o etiquetar un nombre propio en texto en Chino. Texto etiquetado
    mark Representa texto resaltado con propósitos de referencia, es decir por su relevancia en otro contexto. Texto marcado
    ins Define una adición en el documento. Texto agregado
    del Define una remoción del documento. Texto eliminado
